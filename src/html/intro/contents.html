文件目录
使用cli 工具初始化项目 或 手动下载Jet压缩包解压之后，会有如下的文件结构：
但是这两种方式使用略有不同。如下表标注，使用cli 工具初始化项目中会有 build.config.json 文件。
而下载压缩包会有 node_modules ，package.json，package-lock.json 文件。
若是使用标签引入的方式则不用关心这些，只需要引入一个标签即可。
assets (框架相关的代码和资源，无需开发者关注)
node_modules [下载Jet压缩包] (主要是lite-server相关的依赖，无需开发者关注)
src (开发者主要编写代码的目录)
css (css 文件目录)
common.css (全局公共样式)
css.conf (css 配置文件)
html (页面模板和组件的目录)
image (图片资源目录)
js (js文件和模块文件)
media (媒体资源文件)
config.js (在index.html中引用了该文件，该文件是一个全局配置文件，作用是加载依赖库、设置路由、国际化、验证等初始化参数。)
root.html (根组件)
root.js (根组件的js部分，也可将其放入root.html中，该文件非必需)
build.config.json [使用 cli 工具] (项目信息和 jet build命令需要的一些配置信息)
index.html (入口html文件，开发者除了修改head中的信息外，基本无需关注)
package.json [下载Jet压缩包] (项目信息与相关依赖，无需开发者关注)
package-lock.json [下载Jet压缩包] (项目信息与相关依赖，无需开发者关注)
